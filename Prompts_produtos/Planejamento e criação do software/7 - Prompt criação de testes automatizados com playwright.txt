Objetivo
Criar testes automatizados Web para User Story contidas no readme.md

Contexto
- A User Story e seus critérios estão no no readme.md
- As condições de teste estão no arquivo "4 - Prompt criação de condições de teste.txt"
- Você conseguirá os locators da aplicação Web na própria aplicação que está em ./webapp/
- Diferenciação de casos de teste e condições de teste:
  Casos são detalhados, condições de teste são mais genéricas, por exemplo:
   A regra de negócios "Deve ser possível filtrar os agendamentos por período (ex: dia, semana, mês) e por status."
   geraria uma condição de teste "Filtrar dados com base nos filtros disponíveis". A partir de condições de teste podem gerar muitos casos de teste como "Filtrar dados por dia", "Filtrar dados por semana", "Filtrar dados por mês", etc.
   Condições de teste são mais abstratas e casos de teste são mais específicos.


Regras
- Usaremos playwright para escrita dos testes automatizados Web
- Se a estrutura de automação de testes com as ferramentas mencionadas ainda não estiver pronto, crie ela
- Escreva apenas os testes automatizados que cobrirão os casos de teste das user stories
- Gere relatório de execução de testes em HTML
- Crie uma estrutura escalável e bem organizada utilizando padrão de projeto actions fixtures no playwright

Passo a Passo da Execução:
1. Leia as user stories e seus respectivos critérios de aceite que estão no README.md
2. Leia as condições de teste do arquivo "3 - Condições de Teste.txt"
3. Identifique se precisamos instalar as bibliotecas para construção dos testes automatizados Web
4. Converta as condições de teste em casos de teste e grave no arquivo "8 - casos de teste.txt", sempre identificando a que user story e condição de teste o caso de teste se refere.
5. Use os casos de teste do arquivo "8 - casos de teste.txt", que você criou, para então gerar os testes automatizados Web
6. Execute os testes automatizados Web para verificar se foram escritos como deveriam
